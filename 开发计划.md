# 校园地图项目 · 开发计划

## 技术栈选型

**前端框架**：React 18 + TypeScript + Vite

选React而不是原生小程序的原因：一套代码后续可以通过 Taro 或 uni-app 转小程序，初版先做成 PWA（Progressive Web App）部署到服务器，用户通过浏览器或微信内网页访问。TypeScript 在 AI 辅助编码时能提供更好的类型提示，减少低级错误。

**样式方案**：Tailwind CSS 4

不需要手写 CSS 文件，直接在 JSX 里用 class 控制样式。和设计稿的还原效率高，AI 工具生成 Tailwind 代码的准确率也比手写 CSS 好。

**设计规范基准**：`前端设计稿/设计稿SVG/设计指引.svg`

首版先把设计指引里的颜色和常用控件尺寸整理成 CSS 变量，避免开发阶段靠肉眼取色。当前已确认的核心 token 如下：

- 主色：`#1E80C1`
- 主色浅底：`#D7E8F3`
- 主色深底：`#125B8B`
- 页面背景：`#FFFBF2`
- 卡片底：`#FFFFFF`
- 次级底色：`#F1F5F9`
- 主文字：`#0F172A`
- 次文字：`#6F7C8E`
- 分隔线：`#CBD5E1`
- 成功态：`#16A34A`
- 注意态：`#F59E0B`
- 报错态：`#DC2626`

常用控件尺寸也以设计指引为准：校区切换按钮高 `32px`、圆角 `16px`；搜索框高 `41px`、圆角 `20.5px`；"到这去"按钮高 `34px`、圆角 `17px`。

**地图渲染**：内联 SVG + React 组件化

校园地图用 Illustrator 导出的 SVG 直接嵌入页面，优先保留整张 SVG 的 DOM 结构，通过现有的 `<g id="...">` 元素做建筑高亮、点击和定位交互，不额外再造一套建筑 id 命名。`buildings.json` 中的建筑 `id` 与 `svgElementId` 直接复用 SVG 里的现有 id（例如 `main_library`、`jd_library`、`GYM`）。地图的平移和缩放用 CSS transform 实现，不依赖第三方地图库。

**数据层**：静态 JSON 文件

初版不搭后端，所有 POI 数据、校车时刻表数据以 JSON 文件形式放在项目的 `/public/data/` 目录下。后续如果需要动态更新，再接入后端 API，届时只需要把 JSON 的 import 改成 fetch 请求。

**路由**：React Router v6

四个 Tab 页对应四个路由：`/map`、`/shuttle`、`/offcampus`、`/profile`。

**部署**：Vercel 或 Netlify

零配置部署，推送代码自动构建发布，自带 HTTPS。初版用免费额度足够。

## 开发工具

**主力编码工具**：Claude Code（终端 CLI）

适合本项目的原因：直接在终端里跟 Claude 对话，它能读写项目文件、运行命令、调试报错。对于不熟悉 React 的开发者，Claude Code 可以直接生成组件代码并写入文件，比在网页聊天里复制粘贴效率高很多。

使用方式：在项目根目录运行 `claude`，用自然语言描述需求，比如"创建一个 BottomSheet 组件，支持三个吸附高度"，Claude Code 会直接创建文件并写入代码。

**辅助工具**：

- VS Code：代码审查和手动微调，装 Tailwind CSS IntelliSense 插件
- 浏览器 DevTools：移动端调试，Chrome 的 Device Mode 模拟手机屏幕
- Figma（可选）：如果需要从设计稿中精确测量间距和颜色值

**版本管理**：Git + GitHub

每完成一个功能模块就 commit 一次，方便回滚。

## 数据结构定义

### buildings.json 单条记录

```json
{
  "id": "main_library",
  "name": "本部图书馆",
  "campus": "baoshan",
  "category": "图书馆",
  "svgElementId": "main_library",
  "openStatus": "每日工作时段",
  "accessMethod": "刷校园卡",
  "phone": "12345678900",
  "facilities": [
    { "name": "打印机", "available": true },
    { "name": "电梯", "available": true },
    { "name": "贩卖机", "available": false }
  ],
  "photos": ["library-1.jpg", "library-2.jpg"],
  "hasIntro": false
}
```

说明：初版 `buildings.json` 里的 `id` 与 `svgElementId` 保持一致，直接使用地图 SVG 的现有建筑 id，避免维护额外的映射关系。

### shuttle-schedule.json 结构

```json
{
  "sourceFile": "校车时刻表.pdf",
  "version": "2025-09",
  "normalizationNotes": [
    "按线路拆分 from / to",
    "时刻按 weekday / weekend / holiday / winterBreak / summerBreak 五类日期桶归档"
  ],
  "routes": [
    {
      "id": "baoshan-to-jiading",
      "from": "宝山校区",
      "to": "嘉定校区",
      "sourcePage": 1,
      "note": "工作日班次较多，部分班次需预约",
      "schedules": {
        "weekday": [
          {
            "departureTime": "07:00",
            "isReservation": true
          },
          {
            "departureTime": "08:30",
            "isReservation": false
          }
        ],
        "weekend": [],
        "holiday": [],
        "winterBreak": [],
        "summerBreak": [
          {
            "departureTime": "09:00",
            "isReservation": false,
            "viaCampus": "宝山校区"
          }
        ]
      }
    }
  ]
}
```

说明：校车数据直接复用现有 `shuttle-schedule.json` 结构，核心字段为 `id`、`from`、`to`、`note`、`sourcePage`、`schedules`。`schedules` 下统一使用 `weekday / weekend / holiday / winterBreak / summerBreak` 五种日期桶，每个班次至少包含 `departureTime` 和 `isReservation`，需要标记“经停宝山校区”的班次额外使用 `viaCampus`。

### academic-calendar.json 结构

```json
{
  "academicYears": [
    {
      "id": "2025-2026",
      "firstSemester": {
        "start": "2025-09-22",
        "end": "2026-01-25"
      },
      "winterBreak": {
        "start": "2026-01-26",
        "end": "2026-03-01"
      },
      "secondSemester": {
        "start": "2026-03-02",
        "end": "2026-08-02"
      },
      "summerBreak": {
        "start": "2026-08-03",
        "end": "2026-09-13"
      },
      "holidayDates": [
        "2025-10-01",
        "2025-10-02",
        "2025-10-03"
      ],
      "workdayOverrideDates": [
        "2025-09-28",
        "2025-10-11"
      ]
    }
  ]
}
```

说明：除学期区间、寒暑假区间外，其他日期默认按中国标准工作日 / 周末 / 法定休假处理；前端按 `holiday → winterBreak → summerBreak → workdayOverride → weekend → weekday` 的顺序自动判定日期桶。

## 开发日程（14 天）

### 第 1 天：项目初始化 + 骨架搭建

- 用 Vite 创建 React + TypeScript 项目
- 配置 Tailwind CSS
- 实现 AppLayout + TabBar 组件，完成四个 Tab 的路由切换
- 校外 Tab 放占位页（"功能还在更新当中，以后再来吧！"）
- 个人 Tab 放静态页面（头像 + 意见反馈/设置/关于三个入口）

交付物：四个 Tab 可以互相切换，底部导航栏高亮跟随当前页面。

### 第 2-3 天：SVG 地图渲染

- 整理 Illustrator 文件的图层命名，保留现有建筑 `<g id>`，导出 SVG，用 SVGO 压缩
- 实现 CampusMap 组件：加载 SVG、内联渲染
- 实现地图的平移（拖拽 / 触摸滑动）和缩放（双指捏合 / 滚轮）
- 实现 CampusSelector 校区切换下拉，切换时加载对应校区的 SVG
- 清理误留的无用大体积路径元素
- 确保在手机浏览器上手势操作流畅

交付物：地图可以正常显示，支持平移缩放，可以切换校区。

### 第 4-5 天：搜索面板 + 筛选逻辑

- 实现 BottomSheet 组件：三档吸附（收起只露搜索框、半屏、全屏）
- 实现 SearchBar 搜索框 + TagFilter 标签组件
- 编写 buildings.json 测试数据（先填 10-15 栋建筑）
- 实现搜索过滤逻辑：文字搜索（第一层）→ 标签筛选（第二层）
- 实现 SearchResults 列表：建筑缩略图、名称、开放时间、设施标签（深浅色区分可用状态）
- 实现搜索无结果的空状态（含"清除筛选标签"按钮）

交付物：搜索框输入文字可过滤建筑，标签可筛选分类，空状态正常展示。

### 第 6-7 天：地图交互 + POI 详情

- 实现点击地图上的建筑 → 地图居中平移到该建筑 → 底部弹出 POI 详情卡片
- 实现 BuildingLayer 高亮逻辑：
  - 标签筛选时：匹配建筑填充浅蓝色（批量高亮）
  - 点击选中时：该建筑蓝色轮廓线
- 实现 POI 详情卡片：名称、"到这去"按钮（跳转高德地图）、设施标签、开放时间、进入方式、联系电话、照片展示
- 搜索结果列表中点击某条 → 同样触发地图定位 + 详情卡片弹出

交付物：完整的 地图 ↔ 搜索 ↔ 详情 交互闭环。

### 第 8-9 天：校车时刻表

- 实现 ShuttlePage 主页面布局
- 实现日期选择与日期桶自动判定：支持 `weekday / weekend / holiday / winterBreak / summerBreak`
- 新增 `academic-calendar.json`，配置学期区间、寒暑假、法定节假日和调休工作日
- 实现 RouteSelector：出发地/目的地下拉选择器（宝山/嘉定/延长/陈太公寓），带交换按钮
- 接入现有 `shuttle-schedule.json` 数据（含 `sourcePage`、`note`、`viaCampus`）
- 实现时刻表展示逻辑：根据当前时间计算"最近一班"，区分预约/非预约，并支持经停宝山校区标记
- 实现"当日本线其他时刻"展示与当日无班次空状态
- 保留"查看全部时刻表"入口，直接跳转 PDF；乘车点导航、预约网站等扩展能力放到后续版本

交付物：校车时刻表可以按日期自动落到正确的日期桶，并完成线路查询、最近一班和当日本线其他时刻展示。

### 第 10-11 天：数据填充 + 细节打磨

- 补全所有建筑的 JSON 数据（从已有的数据收集表导入）
- 补全建筑照片（压缩后放入项目）
- 检查所有 SVG 建筑的图层 id 与 JSON 数据的 `id` / `svgElementId` 完全一致
- 打磨 UI 细节：间距、字号、颜色与设计指引 token 对齐
- 处理边界情况：超长建筑名的文字截断、照片加载失败的 fallback、网络离线提示
- 添加页面切换动画、BottomSheet 滑动动画

交付物：数据完整、视觉还原度高、边界情况有兜底。

### 第 12-13 天：移动端适配 + 测试

- 在真机上测试（iOS Safari、Android Chrome、微信内置浏览器）
- 修复触摸手势在不同浏览器上的兼容问题
- 检查不同屏幕尺寸的布局适配（iPhone SE ~ iPhone 15 Pro Max）
- SVG 地图在小屏幕上的初始缩放级别调整
- 性能优化：SVG 文件体积检查、图片懒加载、组件按需渲染
- 配置 PWA manifest（图标、启动画面、离线缓存策略）

交付物：真机体验流畅，PWA 可添加到桌面。

### 第 14 天：部署上线

- 推送代码到 GitHub
- 连接 Vercel 或 Netlify 自动部署
- 绑定域名（如有）
- 在不同设备上做最终验收测试
- 在地图页面底部添加免责声明："本地图仅供校内参考，不作为测绘成果"

交付物：线上可访问的 v1.0 版本。

## 给 Claude Code 的使用建议

**项目启动时**，让 Claude Code 一次性生成项目骨架：

```
请用 Vite 创建一个 React + TypeScript 项目，配置好 Tailwind CSS 4 和 React Router v6。
创建四个页面组件 MapPage、ShuttlePage、OffCampusPage、ProfilePage，
用 TabBar 底部导航栏连接。TabBar 有四个图标：地图、校车、校外、我的。
```

**开发组件时**，把设计稿截图直接给 Claude Code 看（它支持读取图片），然后描述：

```
看这张截图，帮我实现这个 BottomSheet 搜索面板组件。
它有三个吸附高度：收起（只露搜索框和标签，约屏幕 20%）、半屏（50%）、全屏（90%）。
用 Tailwind CSS 写样式，支持触摸拖拽切换高度。
```

**遇到报错时**，直接把终端的报错信息贴给 Claude Code，它会自动定位文件并修复。

**每天结束时**，让 Claude Code 做一次代码审查：

```
请检查今天新增的代码，看有没有 TypeScript 类型错误、未处理的边界情况、
或者可以优化的性能问题。
```

## 风险与缓冲

**SVG 地图是最大的不确定性**。Illustrator 图层整理和导出可能比预期花更多时间，尤其是建筑形状和文字标注的分离。如果第 2-3 天发现 SVG 渲染有问题（比如文件太大导致卡顿、建筑点击区域不准确），可以先用简化方案：地图作为静态背景图片，建筑位置用绝对定位的标记点（marker）代替 SVG 元素高亮。这样能保证后续进度不被阻塞。

**BottomSheet 手势交互也有复杂度**。如果自己实现拖拽吸附的物理效果耗时太长，可以引入现成的库（如 react-spring + use-gesture），让 Claude Code 帮你集成。

**如果 14 天不够用**，优先保证的功能是：地图浏览 + 搜索 + POI 详情 + 校车时刻表。可以砍掉的是：建筑轮廓高亮（改用 marker 标记）、BottomSheet 拖拽动画（改用固定两态切换）、PWA 配置（直接用网页访问）。
